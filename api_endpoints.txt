UNIVERSAL ERROR CODES
=====================
  INVALID_CREDENTIALS      401  Bad login
  SESSION_REVOKED          401  Refresh token invalid — force re-login
  ACCOUNT_INACTIVE         403  User deactivated
  ROLE_FORBIDDEN           403  Caller lacks role for this endpoint
  GEOFENCE_VIOLATION       422  Punch outside 200m radius
  ALREADY_PUNCHED_IN       409  Open session exists
  NO_OPEN_SESSION          409  Tried to punch out with no open session
  CROSS_DAY_PUNCH          422  Punch spans midnight — reject
  OVERLAP_DETECTED         409  Manual entry overlaps existing punch
  PERIOD_LOCKED            409  Month is finalized — edit blocked
  BLOCKERS_EXIST           409  Pending verifications stop finalization
  ALREADY_FINALIZED        409  Can't re-finalize
  EMI_EXCEEDS_BALANCE      422  EMI larger than loan balance
  EFFECTIVE_BEFORE_JOIN    422  Wage effective date before join date
  EFFECTIVE_BEFORE_LAST    422  Wage effective date before previous change
  INVALID_TIME_RANGE       422  Punch out before punch in
  MISSING_ENTRIES          422  Some employees missing in finalize payload
  CODE_TAKEN               409  Employee code already exists


CLIENT IMPLEMENTATION NOTES
===========================

1. EMPLOYEE APP lifecycle:
   - On app open → GET /me/today (everything needed in one call)
   - Poll /me/today every 30s while app is foreground
   - Punch button → POST /punches/in or /punches/out
   - Clock is synced from server_time in every /me/today response
   - Silent refresh: when access token < 5 min from expiry, call /auth/refresh

2. ADMIN SALARY SCREEN lifecycle:
   - Open screen → GET /salary/period?year=X&month=Y
   - User types deduction inputs → debounce 400ms → POST /salary/period/preview
     to update Net column live
   - User clicks Finalize → POST /salary/period/finalize with all entries

3. MANUAL PUNCH ENTRY lifecycle:
   - Admin fills form (date, punch-in, punch-out)
   - Before submit → POST /employees/:id/punches/check-overlap
   - If overlap → show warning modal with full_day_history
   - Admin proceeds → POST /employees/:id/punches with override_overlap: true
     if they chose to override

4. SESSION MANAGEMENT (auto-logout fix):
   - Refresh token stored in secure device storage (Keychain/Keystore)
   - Refresh token NEVER expires server-side
   - Client rotates refresh token on every /auth/refresh call
   - Only explicit logout or password change revokes the session
